Merge Request Guidelines#

Note

See the Glossary for the meaning of the acronyms used in this guide.

Please follow these steps to submit a contribution using a merge request:

  • Clone the repository to your local computer:

    # Clone with HTTPS
git clone https://github.com/usnistgov/dioptra.git

# Clone with SSH
git clone git@github.com:usnistgov/dioptra.git

  • Set up a development environment by following the Install instructions in the project README and installing the project’s pre-commit hooks:

    pre-commit install --install-hooks
pre-commit install --hook-type commit-msg

  • Make your changes in a new git branch:

    git checkout -b my-branch master

  • Commit your changes using commit messages that follow the commit message styleguide:

    # NOTE: the optional commit `-a` option will automatically "add" and "rm" edited files
git commit -a

    The pre-commit hooks will run a linter that scans your commit message for formatting errors. Adherence to these guidelines is required because these commit messages are used to automatically generate the project CHANGELOG.

  • Push your branch to GitHub:

    git push origin my-branch

  • In GitHub, open a merge request and set my-branch as the source branch and master as the target branch. If you need to make changes, you can either push them as additional commits to my-branch or use git rebase to amend your initial commits and force push them:

    git rebase master -i
git push origin my-branch -f

  • Once the contribution has been approved, the project maintainer will rebase my-branch on the master branch, generate the updated CHANGELOG, and complete the merge. Please note that the project maintainer may split, squash, and/or reword some of your commits while preserving your authorship of your code changes.
